## 現状の実装(前提知識)

- `treats` フィールドは型だけ存在する: `src/core/types.ts` の `StageDef.treats?: string[]`。
  `src/stage/stageMap.ts` の `defineStage` は文字列をそのまま素通しで、
  パース・検証は未実装
- 経路判定: `src/core/path.ts` の `findPath(grid)`。スタート→ゴールの単純BFSで、
  未完成時は到達できた最遠マスまでの route を返す(首かしげ演出用)
- 判定と失敗表示: `src/game/state.ts` の `startWalk()`。失敗トーストは
  `hud.showToast('あれれ? みちが つながって いないみたい', 2800)`
- お散歩: `src/game/walk.ts` が route(マス列)に沿って移動
- ワールド登録: `src/stage/catalog.ts` の `WORLDS`。w2 は `stages: []` で
  「もうすぐ とうじょう」ロック中。ここに4ステージを入れるだけで選択画面・
  解放条件・クリア記録が動く
- 犬データ: `src/stage/dogs.ts`。`scale` は記念写真の zoom 補正
  (`src/game/photo.ts` の `photoZoomForFriendScale`)にも使われるので、
  新犬種も実際の体格に合わせて設定すること
- ステージ解可能性テストの見本: `tests/w1stages.test.ts`(総当たりソルバ)。
  パーサテストの見本: `tests/stageMap.test.ts`

## 変更内容(この順で。各ステップでテストを書いてから次へ)

1. **core: おやつ対応の経路判定**(`src/core/`、純粋TSのみ)
   - BFSを「(マス, 取ったおやつの集合ビットマスク)」の状態に拡張し、
     全おやつを通ってゴールに至るルートを返す。おやつは最大3個
   - 判定結果は3値に区別できること:
     完成(全おやつ通過ルートあり)/ 道未完成 / 道は完成だがおやつを全部通れない
   - ユニットテスト必須。最低限: おやつなしで従来と同じ挙動 /
     一本道上のおやつ / **分岐があり近道はおやつを通らないが遠回りは通る盤面で、
     遠回りルートが返る** / どう組んでもおやつに届かない盤面
2. **stage: treats のパースと検証**(`src/stage/stageMap.ts`)
   - `'x,z'` を座標にパース。盤外・添景/芝生マス(道になり得ないマス)は
     具体的な日本語メッセージ付きでパースエラー。パーサテスト追加
3. **game/scene: 判定の配線と演出**
   - `startWalk()` の場合分け: 道未完成 → 既存挙動のまま。
     おやつ残り → **お散歩を開始せず**、おやつを強調(ぷるぷる等)+
     トースト「おやつが のこってるよ!」。盤面はそのまま
   - おやつの3D表示: 骨型 or クッキー型をプリミティブで(6.2のパレット内)。
     クリア時のお散歩中、しばちゃんが通ったら「ぱくっ」と消える演出
     (scene/ に判定ロジックを書かないこと。何をいつ消すかは game/ が指示する)
4. **W2の4ステージ**(w2-s1〜w2-s4、難度表どおり 🦴2・3・3・4)
   - w2-s1: おやつ1・空き6 の導入 / w2-s2: おやつ2・空き8 /
     w2-s3: おやつ2を**最短路では取れない配置** / w2-s4: おやつ3・空き10
   - イギリスの街パーツを新添景トークンで追加(例: `レ`=レンガの家、
     `電`=でんわボックス)。M1のトークン表(plan.md)にも追記
   - `tests/w1stages.test.ts` と同様の解可能性テストを4面全部に。
     ソルバはおやつ対応版の判定を使うこと
5. **犬データ4種**: コーギー / ビーグル / ブルドッグ / ゴールデンレトリバー。
   原産国・性格・大きさ・豆知識を正確な情報で、ふりがな付き(UIの漢字は
   `<ruby>` 必須)。`scale` は実際の体格に合わせる
6. **カタログ登録**: `catalog.ts` の w2 に4ステージを設定

## 変更しないこと

  データを足すだけで動くはず。動かない場合は場当たり修正せず報告すること)
- W1のステージと挙動(おやつなしステージは従来どおり)
- design-guide にない色・演出の追加。カメラ角度
