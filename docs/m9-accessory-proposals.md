### 見た目の意図(承認用メモ)

1. **さくらバンダナ**: 要件例「バンダナ」の素直な第一弾。ピンクは既存UI強調色で子供にも分かりやすい
2. **あかいぼうし**: W2の赤いモチーフを頭上に。つば付きでバンダナとシルエット差を出す
3. **あおいリボン**: 帽子・バンダナと被らない「耳アクセ」。アルプス/空色の連想
4. **みどりのスカーフ**: W1と同型で「色違いコレクション」を分かりやすくする(実装コストも低い)
5. **ほしのバッジ**: フィナーレ報酬らしく胸元。黄+オレンジは既存花・ボタン色

### 却下しやすい代替(今回は採用しない案)

- チェック柄・国旗模様 → テクスチャ禁止のため不可
- 鈴・金属光沢 → 発光・PBR禁止
- 大きなマント → お散歩の可読性を落とす

## 3. 報酬・保存・UI(見た目以外・実装方針案)

見た目と独立して先に**合意してよい方針メモ**。ただし **§4 承認前はコード変更一切なし**(文書の修正のみ)。SaveData 拡張も承認後に着手する。

### 3.1 セーブ(`version: 2` 維持)

```ts
// SaveData へ optional 追加。normalizeSaveData でデフォルト補完
ownedAccessories: string[];          // 例: ['acc-w1', 'acc-w2']
equippedAccessoryId: string | null;  // 未装備は null
```

- 既存セーブにフィールドが無くても壊れない
- 破壊的変更・version 3 は不要(AGENTS.md 7章)

### 3.2 報酬トリガー

- `registerAndClear()` 内で、クリア前後の `clearedSet` + `isWorldCleared` を比較
- **そのワールドを初めてクリアしたときだけ**対応 `acc-wN` を `ownedAccessories` に追加
- 再クリア・別プロフィールは既存のプロフィール分離セーブに従う

### 3.3 「もらった!」演出

- クリア画面の直後、またはクリア画面内の追加カードで「〇〇を もらった!」
- 否定・失敗表現は使わない。既存の出会い演出トーンに合わせる

### 3.4 付け替えUIの入口(要選択)

| 候補 | 根拠 | 推奨 |
| ---- | ---- | ---- |
| A. タイトル画面に「きせかえ」ボタン | plan.md M9「タイトルまたはステージ選択」 | **推奨**(到達しやすい) |
| B. ステージ選択画面 | requirements 5.5 | 可 |
| C. プロフィール画面 | requirements 5.5「プロフィール画面や…」 | 可だが編集UIと混ざる |

承認時に **A / B / C** のいずれかを指定する。未指定なら実装は A とする。

### 3.5 3D反映タイミング

- ゲーム起動時・装備変更直後・ステージ開始時にしばちゃんへ再適用
- `createDog` 後にアクセサリーを子メッシュとして着脱できる API を `shiba.ts`(または隣接モジュール)に追加
