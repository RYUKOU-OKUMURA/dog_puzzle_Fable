## M1: ステージ定義のテキスト地図化

20ステージ量産の土台。手書きの `stage01.ts` を、テキスト地図から生成する形式に置き換える。

**仕様(この形式で実装する)**:

- ステージは「地図(文字列配列)+ メタ情報」で定義する

```ts
defineStage({
  id: 'w1-s1',
  name: 'にほんの まち 1',
  world: 'w1',
  encounterDogId: 'akita',
  // 1行=1マス行、トークンは空白区切り。行・列数は自由(最大12×12)
  map: [
    '木 ビ .  家 .  木 ビ 木',
    '花 ★ .  池 .  家 .  木',
    '木 │ 家 .  花 ビ 木 .',
    '花 │ □ 木 │ 家 ビ 木',
    '木 □ ─ ─  □ ─ □ 家',
    '家 .  木 花 □ 木 │ 花',
    '木 ビ .  家 木 花 ◎ 木',
    '花 .  木 .  家 木 鳥 木',
  ],
  treats: [], // おやつのマス 'x,z'(M5から使用)
  palette: ['straight', 'corner', 'tee'], // このステージで使えるパネル種
});
```

- トークン表: `.`=芝生 / `□`=空きスロット / `★`=スタート / `◎`=ゴール / `│ ─ ┌ ┐ └ ┘ ├ ┤ ┬ ┴`=固定道(文字の形=接続方向) / `橋`=固定の橋(上下独立交差。W4〜) / `木 家 ビ 花 池 鳥`=添景 / `レ 電`=添景(W2〜: レンガの家・でんわボックス) / `山 噴`=添景(W3〜: ゆきやま・ふんすい) / `サ カ`=添景(W4〜: サボテン・カラフルな家) / `岩 柵`=障害物添景(M12: いわ・こうじのさく)
- スタート・ゴールの向き(rotation)は、隣接する固定道またはスロットの位置から自動導出する。曖昧な場合(隣接候補が複数)はパースエラーにする
- パーサは `core/` または `stage/` の純粋TSモジュール。不正な地図(★が2個、◎がない、道が盤外を向く等)は**具体的な日本語メッセージ付きで throw** する

**受け入れ条件**:

- [x] stage01 をこの形式に移行し、移行前とゲーム内の見た目・挙動が同一(スロット位置・固定道・添景の完全一致をテストで保証)
- [x] パーサのユニットテスト(正常系+全パースエラー系)
- [x] 既存の `StageDef` を消費する側(town/grid/puzzle)は無変更で動く

## M5: おやつ収集 + ワールド2(イギリス)

**仕様**:

- 経路判定は「(マス, 取ったおやつの集合)」を状態とするBFSに拡張する。スタート→ゴールで**全おやつを通る**ルートが存在すればそれを歩行ルートとして返す(近道があっても、おやつを通る側をえらぶ。requirements 5.1)。おやつが3個以下なので状態数はマス数×8で十分小さい
- 失敗の場合分け: (a) ゴールまで道がつながっていない → 既存の首かしげ+「あれれ? みちが つながって いないみたい」のまま (b) ゴールには届くがおやつを全部通るルートがない → **お散歩を開始せず**、おやつをやわらかく強調表示(ぷるぷる等)+トースト「おやつが のこってるよ!」。盤面はそのまま残す
- おやつを食べる演出はクリア時のお散歩中のみ発生(判定が「全部通るルートのみ歩く」ため、食べ残しは起きない)
- `treats` は `'x,z'` 文字列。`defineStage` で検証し、盤外・添景/芝生マス(道になり得ないマス)はパースエラー
- イギリスの街パーツはテキスト地図の新トークンとして追加(例: `レ`=レンガの家、`電`=でんわボックス)。トークン表とパーサテストも更新する

**受け入れ条件**:

- [x] `treats` に指定したマスを**すべて通る**ルートのみクリア(経路判定は core/ でユニットテスト)
- [x] おやつは3D上に骨型で表示、しばちゃんが通ると「ぱくっ」と消えて食べる演出
- [x] 通れていないおやつがある場合の失敗も優しいフィードバック(「おやつが のこってるよ!」+ おやつ強調)
- [x] W2の4ステージ+イギリスの街パーツ(レンガ家・でんわボックス等。design-guide 6.2 の色域厳守)。難度は難度表どおり(🦴2・3・3・4)。w2-s1 はおやつ1個の導入ステージ
- [x] 犬データ4種(コーギー・ビーグル・ブルドッグ・ゴールデンレトリバー)

## フェーズ2.5: 難易度改修(M10〜M12)

テストプレイで「かんたんすぎる」(盤面が狭い・置く枚数が少ない・パターンが単調)というフィードバックを受けた改修。
新ワールドの追加ではなく、**既存20ステージそのものを大型化・難化**して体験全体を底上げする。
クリア状況はステージIDベースで保存されるため、セーブデータの互換性は保たれる(クリア済みマークはそのまま残り、再挑戦は任意)。

**この改修で変えないこと**:

- 対象年齢と表記ルール(ひらがな分かち書き・ruby)は現状維持。難易度はレベルデザインとデータだけで引き上げる
- 失敗ノーペナルティ・しばちゃんヒントなどの救済設計
- 🦴の配分(1×1 / 2×3 / 3×6 / 4×7 / 5×3)と各ステージの🦴値
- 斜め移動パネルは今回見送り(コアの4方向モデル全面改修になるため。将来のフェーズ3候補)。パネル枚数制限・階段は引き続きスコープ外

M10(基盤)→ M11(ステージ作り直し)→ M12(小ネタ)の順に依存する。M2(Pages)・M9(きせかえ)とは独立。

## M10: 盤面12×12対応の基盤工事

ステージ大型化の前提となるエンジン側の拡張。**このマイルストーンでは既存ステージの見た目・挙動を一切変えない**。
上限を12に留めるのは、iPad縦持ち(820px幅)で12×12ならセル約65pxとタップ操作に余裕が残るため。14以上はパネルが小さくなりすぎるので採らない。

**仕様**:

- パーサ上限 `MAX_SIZE` を 10 → 12 に引き上げる(`stage/stageMap.ts`)
- カメラの表示範囲(`scene/renderer.ts` の `VIEW_SIZE` 固定値)をステージサイズ連動に変える。8×8 は現状の見た目と同一に保ち、大きい盤面だけ引きで映す
- 太陽光のシャドウカメラ範囲(現状 ±8 固定)もステージサイズ連動にし、盤の隅で影が欠けないようにする
- ソルバ(`core/solver.ts`)に枝刈りを追加する(例: 仮置き中でも「残りの空きスロットを全部道とみなしてもゴールに届かない」なら打ち切る到達可能性チェック)。探索予算 200_000 は維持し、12×12・空きスロット20級でも予算内に収まることを実測で確認する

**受け入れ条件**:

- [x] 12×12 の地図がパースでき、13マス以上は従来どおり日本語エラー(境界のユニットテスト)
- [x] 既存 8×8 ステージの画面上の見た目が現状と同一。10×10(w5-s1 / w5-s4)は盤全体が余白付きで収まる。縦長ビューポート(iPad縦 820×1180 相当)で 12×12 が見切れない
- [x] 12×12 の検証用ステージで影・タップ判定・お散歩・記念写真が破綻しない
- [x] 枝刈りが既存全ステージの `isStageSolvable` / ヒントの結果を変えない(回帰テスト)。12×12・空きスロット20の検証用ステージが予算内で解けるユニットテスト
- [x] 既存20ステージの定義は無変更のまま、全テストが通る

> ※ #2・#3 は目視/プレイテストで確認済み(820×1180)。8×8 は camera/影/写真とも現状と同値、大型盤面は連続的に引きで映す。開発時は `__gameDev.bgVerify12()` / `__gameDev.playVerify12()` で再確認できる。

## M11: 既存20ステージの大型化・難化

本丸。難度カーブ(ノコギリ型)を保ったまま全体を底上げする。**ワールド単位(W1→W5)で分割して進めてよい**。
requirements 5.2 の難度表(空き数などの記述)の更新は、各ワールドのステージ変更と同じコミットで行う。

**仕様**:

- サイズと「意図解で実際に置く枚数」の目安:

| ワールド | 現状サイズ | 改修後サイズ                                   | 置く枚数の目安 |
| -------- | ---------- | ---------------------------------------------- | -------------- |
| W1       | 8×8        | 8×8〜9×9(w1-s1 はチュートリアルのため据え置き) | 5〜9           |
| W2       | 8×8        | 10×10 中心(w2-s1 はおやつ導入回なので控えめ)   | 8〜12          |
| W3       | 8×8〜9×9   | 10×10〜11×11                                   | 10〜14         |
| W4       | 8×8〜9×9   | 11×11 中心(w4-s1 は橋導入回なので控えめ)       | 10〜14         |
| W5       | 9×9〜10×10 | 11×11〜12×12                                   | 14〜20         |

- 「ゴールからの逆算」を効きにくくするレベルデザインを各ワールドの後半ステージに入れる:
  - 正解ルートに見えるダミースロット列(偽ルート)
  - 行き止まりで終わる固定道の分岐
