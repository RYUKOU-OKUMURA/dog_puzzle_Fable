# フェーズ2 実装計画 — しばちゃんのおさんぽパズル

[requirements.md](./requirements.md) のフェーズ2を実装可能な単位に分割したもの。
**M1 から順に着手する**(後のマイルストーンは前の成果物に依存する)。
各マイルストーンは「受け入れ条件」を全部満たし、[AGENTS.md](../AGENTS.md) の Definition of Done を通して完了となる。

## M1: ステージ定義のテキスト地図化

20ステージ量産の土台。手書きの `stage01.ts` を、テキスト地図から生成する形式に置き換える。

**仕様(この形式で実装する)**:

- ステージは「地図(文字列配列)+ メタ情報」で定義する

```ts
defineStage({
  id: 'w1-s1',
  name: 'にほんの まち 1',
  world: 'w1',
  encounterDogId: 'akita',
  // 1行=1マス行、トークンは空白区切り。行・列数は自由(最大10×10)
  map: [
    '木 ビ .  家 .  木 ビ 木',
    '花 ★ .  池 .  家 .  木',
    '木 │ 家 .  花 ビ 木 .',
    '花 │ □ 木 │ 家 ビ 木',
    '木 □ ─ ─  □ ─ □ 家',
    '家 .  木 花 □ 木 │ 花',
    '木 ビ .  家 木 花 ◎ 木',
    '花 .  木 .  家 木 鳥 木',
  ],
  treats: [],          // おやつのマス 'x,z'(M5から使用)
  palette: ['straight', 'corner', 'tee'], // このステージで使えるパネル種
});
```

- トークン表: `.`=芝生 / `□`=空きスロット / `★`=スタート / `◎`=ゴール / `│ ─ ┌ ┐ └ ┘ ├ ┤ ┬ ┴`=固定道(文字の形=接続方向) / `木 家 ビ 花 池 鳥`=添景
- スタート・ゴールの向き(rotation)は、隣接する固定道またはスロットの位置から自動導出する。曖昧な場合(隣接候補が複数)はパースエラーにする
- パーサは `core/` または `stage/` の純粋TSモジュール。不正な地図(★が2個、◎がない、道が盤外を向く等)は**具体的な日本語メッセージ付きで throw** する

**受け入れ条件**:

- [x] stage01 をこの形式に移行し、移行前とゲーム内の見た目・挙動が同一(スロット位置・固定道・添景の完全一致をテストで保証)
- [x] パーサのユニットテスト(正常系+全パースエラー系)
- [x] 既存の `StageDef` を消費する側(town/grid/puzzle)は無変更で動く

## M4.1: 図鑑写真の改善 + 文言修正(小規模)

M4レビューで見つかった改善。図鑑の写真が引きすぎて犬種が判別できない問題への対処。

**仕様**:

- 記念写真は**1枚のまま**(出会い画面と図鑑サムネイルで共用)。撮影タイミング(お祝いアニメ後 0.3 秒)も現状維持
- 固定アイソメトリックカメラの角度は変えない(design-guide 7章のカメラ制約厳守)。変えてよいのはターゲット座標・zoom のみ
- ターゲットは現状の「ゴールと友犬の中点」を維持しつつ、zoom を 2.6 から引き上げて2匹が大きく写るようにする(柴犬ポートレートの 4.2 を上限の目安に、2匹がフレームに収まる値を実測で決める)
- 犬種ごとのモデル `scale`(0.72〜1.35)に応じて zoom を補正し、小型犬でも写真内の犬の大きさがほぼ揃うようにする
- 撮影直前に、しばちゃんと友犬の `rotation.y` をカメラ方向(のうち最も近いアイソメ4方位)へ向けてから撮る

**受け入れ条件**:

- [x] 図鑑の 110px サムネイルで、W1の4犬種+柴犬が見分けられる(全犬種で目視確認)
- [x] 出会い画面の大きい写真でも2匹が見切れていない
- [x] `catalog.ts` の W2 説明文「あかい電話ボックス」をかな化(「あかい でんわボックス」)。`sub` は `textContent` 描画のため ruby タグは使えない
