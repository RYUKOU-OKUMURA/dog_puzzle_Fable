まず [AGENTS.md](AGENTS.md) を読み、そのルールに従うこと。仕様の原本は

5.3(しばちゃんヒント)。見た目に触れるので
[docs/design-guide.md](docs/design-guide.md) と [docs/reference/](docs/reference/) も必読。
**M7 完了後に着手すること**(全ギミック=おやつ・palette制限・ダミー・橋 が前提)。

## 背景

最終ワールド。全ギミックを複合した最難関4ステージと、詰まった子供への
救済「しばちゃんヒント」を入れる。難度が最も高い回なので、救済とセットで出すこと。

## 現状の実装(前提知識)

- ソルバ(総当たりで解を探す)はテスト側にしか無い(`tests/w3stages.test.ts` 等)。
  ヒントには「正解ルート上の空きスロット」を実行時に知る必要があるので、
  ソルバを `src/core/` の純粋TSモジュールへ昇格させ、テストはそれを使う形に寄せる
- 失敗の検知: `src/game/state.ts` の `startWalk()` に失敗2分岐がある
  (道未完成の首かしげ / おやつ残り)。ヒントの発火カウントはここに足す
- 無操作の検知: パズル操作は `src/game/puzzle.ts`(選択・配置・回転・除去)を
  経由する。タイマーのリセットはこの操作群にフックする
- しばちゃんの移動・座り: `src/scene/shiba.ts` と `src/game/walk.ts`。
  スロットの強調表示は `src/scene/town.ts` の highlight 系が参考になる
- 図鑑の集計: `src/save/storage.ts` の `countCollected()`。「21種コンプリート」の
  21に**しばちゃん自身を含むかは実装を確認して合わせる**こと
  (出会い犬は 5ワールド×4=20匹。図鑑の総数表示と矛盾しないように)

## 変更内容(この順で。各ステップでテストを書いてから次へ)

1. **core: ソルバ**(`src/core/`、純粋TSのみ)
   - 現在の盤面(配置済みパネル込み)から「クリアできる完成形」を1つ探索し、
     正解ルート上でまだ正しく埋まっていない空きスロットを1つ返す関数
   - palette・おやつ・橋をすべて尊重すること(M6/M7 のソルバテストを流用)
   - ユニットテスト必須: 一意解の盤面で正しいスロットを返す / 既に完成済みなら
     ヒント対象なし / 解なし盤面(誤配置で詰み)でも壊れず「はずすべきパネル」か
     未配置スロットを案内できる、のいずれかを仕様として明記して実装
2. **game: ヒントの発火**(`src/game/state.ts`)
   - 発火条件: パズル中に **90秒無操作** または **お散歩失敗2回**(首かしげ・
     おやつ残りの両方をカウント)。発火は1回につき1マス。操作があれば
     タイマーはリセット。お散歩中・演出中は発火しない
   - 演出: しばちゃんが正解ルート上の空きスロット1つへ歩いて**ちょこんと座り**、
     そのマスがやわらかく光る。**UIボタンは置かない**(世界観に溶かす。requirements 5.3)。
     ペナルティなし
   - 光り方: design-guide のモーション原則(ループ点滅・高速アニメ禁止)に反しない
     こと。「常時ほんのり明るい」など静的な強調を基本にし、きらっと光るのは
     発火時の1回だけにする
3. **W5 の4ステージ**(w5-s1〜w5-s4、難度表どおり 🦴4・4・5・5)
   - w5-s1: **10×10**(最大盤面)、全要素ひかえめ複合 / w5-s2: おやつ+ダミー多め /
     w5-s3: 全部乗せ(橋+おやつ+palette制限+ダミー) / w5-s4: 最大盤面・最難関。
     クリアで完走のお祝い
   - 雪の街・夜の街の変化球: **夜も「あたたかい夜」— 真っ暗にしない**。
     3Dはパステル色域を維持したまま色相・明度をずらす範囲で表現し、
     具体的な配色案は実装前に人間に見せて承認を得ること(design-guide の
     カラーはデザイン変更=承認事項のため)
   - 解可能性テスト `tests/w5stages.test.ts` を4面全部に(core ソルバを使用)
4. **犬データ4種**: シベリアンハスキー / サモエド / シーズー / ダックスフンド。
   正確な情報+ふりがな(`<ruby>` 必須)。`scale` は実体格どおり
5. **コンプリートのお祝い**: 図鑑21種(実装の総数に合わせる)コンプリートで
   特別なお祝い演出。画面を2秒以上覆わない・既存トーンの範囲で
6. **カタログ登録**: `catalog.ts` の w5 に4ステージを設定

## 変更しないこと

- W1〜W4 のステージと挙動、解放条件・セーブ形式(ヒントは何も保存しない。
  クリアフラグ `{cleared}` のまま)
- design-guide にない色・演出の追加(夜の配色は承認を得たものだけ)。カメラ角度
- スコープ外: 階段・水たまり・パネル枚数制限・BGM/効果音
