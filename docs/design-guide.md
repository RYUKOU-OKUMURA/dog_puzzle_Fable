# デザイン仕様書 — しばちゃんのおさんぽパズル

このドキュメントは**規範**である。「参考」ではない。
ここに書かれたトークン・数値・ルールから外れる見た目の実装は不可。
新しい色・寸法・モーションが必要になった場合は、勝手に足さずタスクの成果物として「提案」を書き、人間の承認を得ること。

現行UI(フェーズ1)のクオリティが**最低ライン**。新しい画面を作るときは、既存のタイトル画面・出会いカード・図鑑と並べて違和感がないことが合格条件。

## 0. ビジュアルリファレンス(目で見る規範)

**[reference/baseline-w1-town.png](reference/baseline-w1-town.png) が最低基準クオリティーの見本**。3Dシーンに触れたら必ずこの画像と実行画面を並べて比較すること。運用ルールは [reference/README.md](reference/README.md)。

## 1. デザインの人格

- **世界観**: 晴れた日の午後のお散歩。パステル・丸み・木のぬくもり
- **トーン**: やさしい・あたたかい・ちょっとおいしそう(クリーム、いちごミルク、みかん色)
- **禁止**: 原色のビビッドカラー、黒(#000)、シャープな角、速い点滅、ダークテーマ、怖い表現

## 2. カラーパレット(UI)

`src/style.css` の CSS変数がすべて。**HEX直書き禁止、必ず変数を使う**。

| 変数              | 値        | 用途                                       |
| ----------------- | --------- | ------------------------------------------ |
| `--sky`           | `#bde3ff` | 背景の空色。3Dシーン背景と一致させる       |
| `--cream`         | `#fff9ee` | カード・パネルの地色                       |
| `--pink`          | `#ffd3de` | やわらかい強調(図鑑ヘッダー、選択状態の地) |
| `--pink-strong`   | `#ff8fad` | ピンク系ボタン、選択枠                     |
| `--green`         | `#bfe8b7` | 補助(自然・OK系)                           |
| `--orange`        | `#ffb066` | 補助オレンジ                               |
| `--orange-strong` | `#f2903d` | 主ボタン(プライマリアクション)             |
| `--brown`         | `#6b4f3a` | テキスト基本色。**黒は使わない**           |
| `--brown-light`   | `#a08468` | 弱いテキスト・控えめボタン                 |
| `--white`         | `#ffffff` | ボタン文字・白フチ                         |

- 影は必ず `--shadow`(`0 4px 12px rgba(107, 79, 58, 0.25)`)。**影は茶色。黒い影は禁止**
- 半透明の白地は `rgba(255,255,255,0.9)`(パレット等)、画面の暗転は `rgba(107,79,58,0.35)`(茶色の暗転)
- 新画面で色が足りない場合も、まずこの10色+透明度違いでやりくりする

## 3. タイポグラフィ

- フォントスタック(変更禁止): `'Hiragino Maru Gothic ProN', 'Zen Maru Gothic', 'BIZ UDPGothic', 'Yu Gothic', 'Meiryo', sans-serif`(丸ゴシック優先。Webフォント読み込みは禁止)
- 見出し・ボタンは `font-weight: bold`
- 文字サイズの基準:

| 用途              | サイズ                                                                     |
| ----------------- | -------------------------------------------------------------------------- |
| タイトルロゴ      | `clamp(34px, 7vw, 64px)` + 白フチ(text-shadow 4方向 3px)                   |
| カード見出し (h2) | 26px                                                                       |
| 大ボタン          | 26px / 通常ボタン 20px                                                     |
| 本文              | 16px、`line-height: 1.8〜1.9`                                              |
| 補足              | 14px。**14px未満の文字を作らない**(パレットボタンのラベル11pxのみ既存例外) |

- **表記ルール**: UIテキストはひらがな・カタカナ中心。分かち書き(「みちを つないで」のように文節で半角スペース)。漢字は犬種名・国名など図鑑的な語のみ、必ず `<ruby>秋田犬<rt>あきたけん</rt></ruby>` 形式。`ruby rt` は 0.45em
- 絵文字はアクセントとして使ってよい(🐾 📖 🎉 💗 ⭐ など。既存の使用箇所とトーンを合わせる)

## 4. 形状・コンポーネント

- 角丸: カード類 `--radius`(20px)、ボタン・バッジ・トーストは**ピル形**(`border-radius: 999px`)、カード内の入れ子要素は 10〜16px。**角丸なしの矩形をUIに出さない**
- ボタン(`.btn`)は既存クラスを使う。新種のボタンを作らない:
  - 主アクション: `.btn`(オレンジ) / 大事な場面は `.btn.btn-big`
  - かわいい系アクション(図鑑・登録): `.btn.btn-pink`
  - 戻る・控えめ: `.btn.btn-sub`(白地・茶文字)
  - 押下時は `scale(0.94)`。**タップターゲットは最小 48px、標準 56px(min-height)**
- カードは `.card`(クリーム地 + `--shadow` + 20px角丸、`max-width: min(520px, 92vw)`)
- 写真は白フチ(`border: 6px solid var(--white)`)+ 角丸 16px の「チェキ風」
- 一時メッセージは `.toast`(白ピル、上から 84px、`toast-pop` アニメ)
- 下端に張り付くUIは必ず `env(safe-area-inset-bottom)` を padding に含める(iPadホーム画面起動対応)

## 5. モーション

- 原則: **短く・ぽよんと・1回だけ**。ループする点滅や高速アニメは禁止
- インタラクション反応は 0.1s(ボタン押下)、出現系は 0.25〜0.35s の ease-out + scale(`toast-pop` 参照)
- ごほうび演出(ハート等)は 1.6〜2.2s で消える。画面を覆い続けない
- 3D側のキャラモーションは「テクテク上下の弾み」「首かしげ」「よろこびジャンプ」系。`game/walk.ts` のトーンに合わせる

## 6. 3Dアートディレクション

### 6.1 マテリアルとライティング(変更禁止)

- マテリアルは **`MeshLambertMaterial` の単色のみ**。テクスチャ・グラデーション・PBR(Standard/Physical)・発光は禁止
- シーン背景: `0xbde3ff`(= `--sky` と同値。UIと地続きの空)
- ライト構成: `AmbientLight(0xffffff, 0.85)` + `DirectionalLight(0xfff3e0, 1.7)` position(6,12,4)。影は `PCFSoftShadowMap`、mapSize 2048
- カメラ: `OrthographicCamera`、オフセット (10,10,10) の固定アイソメトリック。カメラ操作UIを足さない

### 6.2 3Dカラーパレット

| 用途                 | 値                                                         |
| -------------------- | ---------------------------------------------------------- |
| 芝生(市松)           | `0xa9dd9d` / `0x9ed394`                                    |
| パネルスロット(砂地) | `0xf0e2b6`、ハイライト時 `0xffc9d8`                        |
| ジオラマ台座         | `0xd9b98c`                                                 |
| 家の壁               | `0xfff3dd` `0xffe3e9` `0xe4f3ff` `0xf3ffe0` から hash 選択 |
| 屋根                 | `0xe0604f` `0xf2903d` `0x7fbf7a` `0x6faed9` から hash 選択 |
| 木の葉               | `0x69bd63` `0x7fcf74` `0x5cb168`                           |
| 幹・ドア             | `0x9c6b4a`                                                 |
| 花                   | `0xff8fad` `0xffd166` `0xffffff` `0xff6f91`                |

- 新ワールド(イギリス・メキシコ等)の街パーツは、**この明度・彩度帯(パステル)を維持したまま色相だけ土地らしく**ずらす。例: レンガは `#e0604f` 系のくすみ赤、サボテンは既存 TREE_GREENS の範囲。真っ赤・真っ青は禁止
- W5 変化球(`StageDef.theme`): 空(`0xbde3ff`)は変えない。地面の市松だけ既存パレット内でずらす — snow=`#ffffff`/`#e4f3ff`、night=`#ffe3e9`/`#fff3dd`(あたたかい夜・真っ暗禁止)、chinatown=`#7fcf74`/`#a9dd9d`、festival=`#ffe3e9`/`#f3ffe0`
- バリエーションは乱数でなく **座標hash**(`town.ts` の `hash()`)で決める。リロードで見た目が変わらないこと

### 6.3 プロポーション(ジオラマ感の生命線)

- 1マス = 1.0 ワールド単位。地面タイルは `0.98 × 0.12 × 0.98`(2%の目地がジオラマ感を作る)
- 建物はマス内に収める(幅・奥行き ≤ 0.72)。高さは家 ~0.6、ビル ~0.9〜1.4 まで。**高さ1.5を超える構造物は作らない**(盤面が見えなくなる)
- 屋根は四角錐(`ConeGeometry(r, h, 4)` を 45° 回転)がこの世界の「家の記号」。踏襲する
- ディテールは「ドア1枚+窓2つ」程度のミニマムで止める。作り込みすぎるとローポリの統一感が壊れる
- 動的オブジェクト(犬・建物)は `castShadow`、地面は `receiveShadow`

### 6.4 犬モデル

- `scene/shiba.ts` の作りが基準(プリミティブ組み合わせのデフォルメ)。新犬種は **毛色(`furColor`)とスケール(`scale`)+ 最小限の形状差分**(耳の垂れ・マズルの長さ等)で表現する
- 犬をリアル寄りにしない。「ぬいぐるみがそのまま歩いてる」感を守る

## 7. 画面レイアウトの原則

- 3Dシーンが主役。DOM UIは端(HUD)か中央のカード1枚。**画面を同時に覆うUIは1系統まで**
- パズル中の常設UIは「上: ステージ名バッジ+補助ボタン」「下: パネルパレット+スタートボタン」の2段だけ。左右に浮遊ボタンを増やさない
- 全画面系(タイトル・選択画面)は `screen-sky` グラデ(空→夕焼けクリーム)、モーダル系は `screen-dim`(茶の暗転)+ `.card`
- 縦長(iPad縦持ち)・横長どちらでも: 盤面全体が見える(renderer.ts が短辺フィット済み)、UIが盤面の中心を隠さない、`flex-wrap` で溢れ対応
