# フェーズ3 実装計画 — しばちゃんのおさんぽパズル

[roadmap.md](./roadmap.md) 6章の「あそび心地の改善」を、実装可能なマイルストーンに正式化したもの(M13〜)。
[plan.md](./plan.md) はフェーズ2・2.5(M1〜M12)で完結済みのため、**M13以降は本書にのみ書く**。
各マイルストーンは「受け入れ条件」を全部満たし、[AGENTS.md](../AGENTS.md) の Definition of Done を通して完了となる。

着手時は roadmap 6章の導入方針どおり、**1件ずつ実装し子供の反応を見てから次へ**進む。

## M13: こうかおん(効果音3点 + おと ON/OFF)

追加効果音はスコープ外。BGM は **M14** で正式化。今回は効果音3点のみ先行し、子供の反応を見る。

### 仕様

#### 効果音3点 — Web Audio API 合成のみ

音色・パラメータは [prototypes/audio-demo.html](./prototypes/audio-demo.html) から移植する(**勝手に変えない**)。

| 名前           | タイミング                                 | プロトタイプ  |
| -------------- | ------------------------------------------ | ------------- |
| ポン           | パネルをスロットに置いたとき               | `SFX.pon`     |
| ぱくっ         | お散歩中、しばちゃんがおやつを通過した瞬間 | `SFX.paku`    |
| ファンファーレ | クリア成功の演出開始時                     | `SFX.fanfare` |

共通:

- 初回ユーザータップで AudioContext を生成・resume(iPad 自動再生制限対応)
- DynamicsCompressor 経由で音割れ防止。プロトタイプの master 構成を踏襲
- 音声ファイル・howler.js 等の**依存追加は禁止**
- 回転音・失敗音などの追加もしない(3点で子供の反応を見るため)

#### おと ON/OFF 設定

- トグルは1個、名前は「おと」。初期値 ON
- プロフィール別に保存(`save/` の既存形式への**追記フィールド**。旧セーブは欠損時 ON で読める後方互換。破壊的変更禁止)
- UI: タイトル画面に小さなトグルボタン(例: 🔊「おと ON」/ 🔇「おと OFF」)。文言はひらがな。design-guide の既存パレット・ボタン様式のみ使用、新しい色・装飾を足さない
- この構成(タイトル画面・トグル1個・ひらがな文言)までは承認済み。逸脱したくなったら実装せず人間に確認

#### アーキテクチャ

- 新モジュール `src/audio/`(例: `sfx.ts`)。他層に依存しない・DOMも触らない。「鳴らす関数」と有効フラグだけを公開
- 呼び出しは `game/state.ts` の状態遷移と `main.ts` の配線に集約。`scene/`・`core/` から直接呼ばない
- ON/OFF分岐は audio モジュール内で持ち、呼び出し側に散らさない

### 受け入れ条件

- [x] AGENTS.md 6章の Definition of Done 全項目(build / test / lint、通しプレイ、縦長 820×1180)
- [x] プロフィールを切り替えると音設定が正しく追従する
- [x] 旧形式セーブデータの読み込みで音設定が ON になる(後方互換のテストを書く)

## M14: BGM(現在は全ステージ共通1曲 + おさんぽアレンジ)

ワールド別BGM・効果音の追加制作はスコープ外。現在は全ステージで同じ1曲を流し、おさんぽ成功時だけアレンジを重ねる。後からステージごとに曲を追加できるよう、曲データと再生制御を分離し、ステージ定義から曲IDを選ぶ基盤までは含める。

### 仕様

#### BGM — Web Audio API 合成のみ

曲データ・音色は [prototypes/audio-demo.html](./prototypes/audio-demo.html) の W1「にほん」わふうループから移植する(**勝手に変えない**)。

| 項目 | 内容                                                                                              |
| ---- | ------------------------------------------------------------------------------------------------- |
| 曲   | ヨナぬき音階・88BPM・8小節ループ(MELODY / CHORDS / BASS / ARP)                                    |
| 音色 | プロトタイプの `bell` / `kick` / `hat` / delay master 構成を踏襲                                  |
| 依存 | 音声ファイル・howler.js 等の**依存追加禁止**                                                      |
| 共有 | M13 の AudioContext / master / compressor / `soundEnabled` を共用(二重に AudioContext を作らない) |

#### 再生ルール(場面)

| 場面                                                         | BGM                                                                       |
| ------------------------------------------------------------ | ------------------------------------------------------------------------- |
| タイトル / プロフィール選択 / ワールド選択 / ステージ選択    | 通常ループ(アレンジなし)                                                  |
| パズル中(全ステージ共通)                                     | 通常ループ                                                                |
| お散歩中(`phase === 'walk'` かつクリア成功ルート歩行中)      | **おさんぽアレンジ ON**(kick/hat 重ね)                                    |
| 道未接続・おやつ残りの失敗お散歩                             | **通常ループのまま**(アレンジなし・止めない。怖い無音にしない)            |
| 出会い(`encounter`)・クリア画面(`clear`)・きせかえ報酬カード | **BGM停止**。ファンファーレは既存 `playFanfare` のみ(新規SFXを増やさない) |

#### おと ON/OFF

- M13 の `soundEnabled` を共用。OFF なら効果音も BGM も出さない
- トグルUIは増やさない(タイトルの「おと」1個のまま・分離しない)
- ON に戻したとき、現在の画面に応じた BGM を再開する
- プロフィール切替で `soundEnabled` に追従する(M13 後方互換を壊さない)

#### アーキテクチャ

- `src/audio/` に BGM 用モジュールを追加(例: `bgm.ts`)。共通の context 層にまとめてもよいが、**二重に AudioContext を作らない**
- 曲固有の音階・スケジューラは `src/audio/tracks/` に置き、再生制御と分離する
- 各 `StageDef` は `bgmTrackId` を必須で持つ。未登録IDは無音にせず既定曲へフォールバックする
- BGM の残響は BGM 専用 bus の配下へ戻し、停止・切替時に dry / wet をまとめて短くフェードする
- audio 層は他層・DOM に依存しない。「再生モード切替」「停止」「unlock」だけ公開
- 呼び出しは `game/state.ts` の状態遷移と `main.ts` の配線に集約。`scene/`・`core/` から直接呼ばない
- ループは途切れないこと(プロトタイプの schedule/queue 方式を移植)。画面遷移でプチッと鳴らないよう、必要なら短いフェード(やりすぎない。新SEは足さない)
- Safari 無音対策: M13 同様、resume 完了後に schedule する

### やらないこと(スコープ外)

- ワールド別・ステージ別の新曲制作(選曲基盤のみ実装)
- BGM 用の新しいトグル / 音量スライダー
- 効果音の追加(回転・失敗・ワン 等)
- お散歩ランダムイベント・あしあと等の他アイデア
- 50ステージ化

## M15: お散歩ランダムイベント

成功お散歩中に低確率で小ネタ演出を差し込む。純演出のため収集要素・セーブ変更なし。

### 仕様

#### 3種のイベント(見た目オブジェクト付き・事前承認済み)

| 種類 | 見た目 | しばちゃんの動き |
| ---- | ------ | ---------------- |
| ちょうちょ | 胴 Sphere + 羽2枚 Box(花色 `0xff8fad` / `0xffd166`)。前方〜斜め上をひらひら | 短く一回転 |
| 池で立ち止まり | 新規オブジェクトなし(既存 `pond` 添景) | 池の方を向き、約0.7秒静止 |
| 鳥にやさしくワン | 胴 Sphere(`0xfff3dd`) + くちばし Cone(`0x9c6b4a`) + 羽 Box | 首を少し上げて2回こくん。**新SEなし** |

マテリアルは `MeshLambertMaterial` 単色のみ(design-guide 6.2 既存パレット)。怖い・乱暴な動き禁止。

#### 発生ルール

- **成功お散歩のみ**(道未完成・おやつ残りでは出さない)
- **1回の散歩で最大1イベント**
- 発生率 **約18%**(ルート開始時に1回だけ抽選)
- 種類: ルート上に池隣接セルがあればそのセルで `pondStop` 優先。なければ `butterfly` / `bird` を等確率
- 配置セルはスタート・ゴール以外の中間マス。おやつマスと重なっても可(先に食べる演出、その後イベント)

#### アーキテクチャ

- `src/scene/walkProps.ts` — ちょうちょ・鳥のプリミティブ(描画のみ)
- `src/game/walkEvents.ts` — 抽選・種類選択・演出シーケンス
- `src/game/walk.ts` — 軽量エモート(`spinOnce` / `gentleBark`)
- 呼び出しは `game/state.ts` の成功 `walkAlong` の `onArrive` に集約
- 一時オブジェクトは `sceneContext.scene` に add/remove + `disposeObject`
- パズル判定・経路探索・BGM/SE には触れない

- 新SE / BGM変更
- 失敗お散歩へのイベント
- セーブ形式変更
- 50ステージ化・あしあと・スタンプ・図鑑クイズ

- [x] AGENTS.md 6章の Definition of Done 全項目(build / test / lint、通しプレイ、縦長 820×1180)
- [x] 成功お散歩で低確率に3種のいずれかが起きる(1散歩1回まで)
