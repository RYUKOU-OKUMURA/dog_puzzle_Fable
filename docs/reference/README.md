# docs/reference — ビジュアルリファレンス

**「最低基準クオリティー」の見本画像**を置くフォルダ。
[design-guide.md](../design-guide.md) が文章による規範、ここは目で見る規範。両方セットで使う。

## 使い方(エージェント向け)

- 見た目に関わる実装をしたら、実行画面をこのフォルダの基準画像と**並べて比較**する
- 「基準画像と同じゲームに見えるか」が合格ライン。下回る見た目でタスクを完了にしない
- 比較の観点: パステルの色域 / タイルの目地とジオラマ台座 / 影の柔らかさ(茶色系・ソフト) / 屋根=四角錐の記号性 / 添景の密度(詰め込みすぎない)

## concepts/ サブフォルダ

`concepts/` には画像生成AIによる各ステージのコンセプトアートを置く(生成指示は [../concept-art-brief.md](../concept-art-brief.md))。
**concepts/ は発想材料であり規範ではない**。合格判定に使うのは reference/ 直下の baseline 画像のみ。

## 運用ルール

- 新ワールドの街並みが人間の承認を得たら、そのスクリーンショットを `baseline-w2-town.png` のように追加し、上の表を更新する(以降そのワールドの基準になる)
- 基準画像の削除・差し替えは人間のみが行う。エージェントは追加提案まで
- スクリーンショットに個人情報(プロフィール名など)が写り込んでいないこと
