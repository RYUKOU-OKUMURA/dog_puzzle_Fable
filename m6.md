# タスク: M6 パネル種制限(palette)の有効化 + ワールド3(フランス・スイス)

まず [AGENTS.md](AGENTS.md) を読み、そのルールに従うこと。仕様の原本は
[docs/plan.md](docs/plan.md) の「M6」セクション(受け入れ条件)と
[docs/requirements.md](docs/requirements.md) 5.2(難度表: W3-1〜W3-4 の行)。
見た目に触れるので [docs/design-guide.md](docs/design-guide.md)(特に 6.2 の
3Dカラーパレット)と [docs/reference/](docs/reference/) の見本画像も必読。

## 背景

現状のゲームは W1・W2(易しい側)しか無く、子供には簡単すぎる。M6 は
**新ギミックなし**で、眠っている難度レバー2つ(パネル種制限・ダミースロット)と
レベルデザインで「W2 より明確に難しい」W3 を作る回。

## 現状の実装(前提知識)

- `palette` は**定義だけ存在し、どこからも消費されていない**:
  - 型: `src/core/types.ts` の `StageDef.palette?: PanelKind[]`
  - `src/stage/stageMap.ts` の `defineStage` は `input.palette` を無検証で素通し
  - 全8ステージが `['straight', 'corner', 'tee']`(全種)を宣言している
  - HUD のパレットボタンは `PLAYER_PANEL_KINDS`(`src/core/panel.ts`)固定で生成
    (`src/ui/hud.ts` のコンストラクタ)。ステージごとの出し分けは無い
  - 配置側(`src/game/puzzle.ts` の `handleCellTap` → `grid.place`)も種類を検証しない
- HUD はゲーム起動時に1回だけ構築される。ステージ切替時は `updateStageName()` を
  `src/game/state.ts` が呼ぶ(パレット出し分けも同じ流れに乗せるのが自然)
- ダミースロットの前提(M5.1 で確立): 「隣接に道/スロット/★/◎が無い完全孤立スロット」は
  禁止。`tests/stageSlots.test.ts` が **catalog 登録済みの全ステージ**を自動検査するので、
  W3 を `catalog.ts` に登録すれば新ステージも自動で対象になる
- 解可能性テストの見本: `tests/w2stages.test.ts`(ステージごとの総当たりソルバ)。
  パーサテストの見本: `tests/stageMap.test.ts`
- 犬データ: `src/stage/dogs.ts`。`scale` は記念写真の zoom 補正
  (`src/game/photo.ts`)にも使われるので実際の体格に合わせること

## 変更内容(この順で。各ステップでテストを書いてから次へ)

1. **stage: palette の検証**(`src/stage/stageMap.ts`)
   - `defineStage` で検証: 空配列・重複・`PLAYER_PANEL_KINDS` に無い種類は
     具体的な日本語メッセージ付きでパースエラー。未指定(undefined)は全種扱いのまま可
   - パーサテスト追加(正常系+全エラー系)
2. **core: 配置の防御**(`src/core/grid.ts`)
   - `place()` は `stage.palette` に無い種類を拒否する(盤面を変えない)。
     UI から到達不能でも、層の防御として core に置く。ユニットテスト必須
3. **ui/game: パレットの出し分け**
   - HUD にステージの palette を反映するメソッドを追加し(例: `updatePalette(kinds)`)、
     ステージ開始時に `state.ts` から呼ぶ。**palette に無い種類のボタンは非表示**にする
     (グレーの押せないボタンは「押せるのに押せない」と子供が混乱するので出さない)
   - ステージ切替時に選択中パネルは必ず解除(`selectKind(null)`)
4. **W3 の4ステージ**(w3-s1〜w3-s4、難度表どおり 🦴3・3・4・4)
   - w3-s1: **ダミースロット導入**(ダミー2)。ダミー以外は W2 後半と同水準
   - w3-s2: **パネル種制限**(palette から `tee` を外す)+ダミー3。T字が無いと
     分岐でごまかせないので、どこで曲がるかを先読みさせる設計にする
   - w3-s3: おやつ2+ダミー3の複合
   - w3-s4: **9×9**(初の8×8超え)、最長ルート+ダミー4
   - ダミーの質: 「一目で行き止まりと分かるおとり」は効かない。**正解ルートと同じくらい
     もっともらしい偽ルート**(長さが近い・途中まで正解と共有する等)を作ること
   - 石畳・アルプス風の街パーツを新添景トークンで追加(例: `山`=ゆきやま、`噴`=ふんすい)。
     plan.md M1 のトークン表にも追記。色は design-guide 6.2 の色域厳守
   - 解可能性テスト `tests/w3stages.test.ts` を4面全部に。**ソルバは palette を尊重する
     こと**(w3-s2 を T字ありで解いて「解ける」と誤判定しない)。加えて w3-s2 は
     「palette 外の種類を使わないと成立する解が存在する」ことを確認する
5. **犬データ4種**: プードル / パピヨン / セントバーナード / バーニーズマウンテンドッグ。
   原産国・性格・大きさ・豆知識を正確な情報で、ふりがな付き(UIの漢字は `<ruby>` 必須)
6. **カタログ登録**: `catalog.ts` の w3 に4ステージを設定

## 変更しないこと

- パネルの**枚数**制限(スコープ外。導入しない)
- W1・W2 のステージと挙動、解放条件・セーブ形式・図鑑・記念写真のロジック
- design-guide にない色・演出の追加。カメラ角度
