## 5. 50ステージ化企画(ドラフト・人間の承認待ち)

### 5.1 全体構成

**新ワールド W6〜W12(7ワールド × 4ステージ = 28)+ ボーナス2ステージ(W1・W4 に各+1)= 30追加 → 合計50**。
図鑑は柴犬含め **51犬種** になる。

採用理由:

- 盤面上限12×12は変えられないため、新ワールド追加が「地理を学べる国を増やす」という設計原則と両立する自然な拡張軸
- ワールド単位の追加はノコギリ型難度カーブ(ワールド頭で1段易しく)を保ちやすい
- 7×4=28 だけでは48止まりのため、残り2つは「本物の犬種が既存ワールドに余っている場所」に寄せた(5.4節)

ワールド解放順(既存ルール「前ワールド全クリアで解放」を継続):

```
W1〜W5(既存) → W6 ドイツ・オーストリア → W7 イタリア → W8 スペイン・ポルトガル
→ W9 きたヨーロッパ(息ぬき回) → W10 中国 → W11 オーストラリア → W12 とおい せかいの たび(フィナーレ)
```

**⚠️ 実装上の注意(検証済み)**: 「s1〜s4全クリアでボーナス解放・次ワールド解放条件には影響しない」という挙動は、**現行の解放ロジックでは実現できない**:

- `src/stage/progress.ts` の `isStageUnlocked` は「s1クリアで s2以降を解放」のため、s5 も s1 クリアだけで開いてしまう
- 同 `isWorldUnlocked` は「前ワールド**全ステージ**クリア」判定のため、s5 を足すと次ワールドの解放条件にボーナスが混ざる

→ 対処: ステージ定義に `bonus` フラグを追加し、(a) ボーナスは「同ワールドの非ボーナス全クリア」で解放、(b) `isWorldCleared`(ワールド解放判定)からボーナスを除外、(c) `clearCelebration.ts` のフィナーレ判定(3.4節の動的導出)もボーナスを除外する。規模S・ユニットテスト必須。

### 5.6 新ギミック候補(すべて人間の承認事項)

改修コストの小さい順:

1. **経由ポイント演出のワールド別スキン(極小)**: M12 予定の「おともだち犬にあいさつ」機構の見た目差し替え(W7=ジェラート屋台に寄り道、W10=屋台の点心を味見、W11=水飲み場で一休み等)。ロジック変更ゼロ
2. **障害物添景の国別バリエーション(小)**: M12 の岩・工事柵を国ごとに再スキン(W9=雪の吹きだまり、W10=竹足場、W12=砂丘)。判定ロジックは同一
3. **一方通行(矢印)パネル(中〜大)**: 通行方向が固定の新パネル種。`core/` の接続判定・ソルバに方向性を持たせる改修が必要な**唯一の新メカニクス**。導入するなら W12 限定を提案。スコープ外リスト(階段・水たまり・枚数制限・斜め移動)には含まれないが、必ず個別承認+`core/` ユニットテストを伴うこと

### 5.7 リスクと留意点

- **犬種情報の正確性**: 原産国・性格・大きさ・豆知識は信頼できる情報源(FCI等)と突き合わせる。曖昧な逸話(諸説ある起源年代など)は載せない
- **きせかえ報酬の頭打ち**: 3.3 参照。M9 着手前に W6 以降の報酬方針を決める
- **図鑑UIの一覧性**: 51件はワールド(大陸)ごとの見出しでセクション分けが要る。新しい色・角丸・影は増やさず既存 `.card` トークン内で
- **難度インフレ**: 盤面サイズ頭打ちのため密度で難度を作ると5〜10分制限を超えやすい。W9 の息ぬき回・「物量でなく紛らわしさ」方針・テストプレイでのスロット削減で吸収する
- **実装順序**: M12(おやつ5個・障害物)→ 本拡張、の順を推奨(W10〜W12 の設計が M12 前提のため)

## 8. この文書の運用とドキュメント構成

### 8.1 実装再開の手順(次のセッションはここから)

1. 3章の優先順位で着手タスクを決める。M9・M12 の仕様と受け入れ条件は [plan.md](./plan.md) にある(そのまま着手可)

3. 完了判定は常に AGENTS.md 6章の Definition of Done

### 8.2 詳細をどこに書くか(役割分担)

| 場所                        | 書くもの                                                                                         |
| --------------------------- | ------------------------------------------------------------------------------------------------ |
| 本書(roadmap.md)            | 企画の**索引と状態**(承認待ち / 承認済み / 移管済み)。仕様レベルの記述は書かない                 |
